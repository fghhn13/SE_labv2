## 核心概念说明

本文件以简单、面向实践的方式解释本项目中常用的一些核心概念。  
这些概念是后续架构设计与实验讨论的基础。

---

### 1. 环境（Environment）

- **含义**：
  - 环境是智能体交互的世界，定义了：
    - 状态空间（例如栅格地图上的位置分布）；
    - 动作空间（例如上下左右移动）；
    - 状态转移规则（移动、碰壁、到达终点等）；
    - 奖励函数（如到达终点给正奖励、每一步给小负奖励等）。
- **在本项目中的作用**：
  - 提供 2D 栅格最短路径任务的具体实现。
  - 对外暴露统一接口（如 `reset()`、`step(action)`），不负责任何学习或训练细节。

---

### 2. 栅格地图（Grid Map）

- **含义**：
  - 一个由离散格子（cell）组成的二维网格，每个格子可能是：
    - 可通行区域；
    - 障碍；
    - 起点（Start）；
    - 终点（Goal）。
- **在最短路径任务中的作用**：
  - 决定智能体的可行路径与任务难度。
  - 不同地图结构（如迷宫、走廊、开放区域）可能诱导出不同的“涌现结构”。
- **实现相关说明**：
  - 栅格地图可以是手工定义，也可以由地图生成器自动生成（例如随机障碍、迷宫算法等）。

---

### 3. 智能体（Agent）

- **含义**：
  - 在每个时间步，根据环境提供的观测（以及可选的内部状态）选择动作的决策单元。
- **在本项目中的作用**：
  - 承担“在环境中如何行动”的决策任务。
  - 可以是：
    - 非学习型（固定规则、随机策略、简单启发式）；
    - 学习型（通过数据更新参数，例如强化学习）。
- **设计原则**：
  - 保持接口简洁（例如 `act(observation, state)`）；
  - 避免将整体训练流程塞入 Agent 内部，训练循环由 Trainer 负责。

---

### 4. 训练器（Trainer）

- **含义**：
  - 负责组织训练 / 采样过程的控制器，拥有 episode 与 step 循环的控制权。
- **在本项目中的作用**：
  - 驱动环境和 Agent 的交互：
    - 重置环境、运行步骤、检测 episode 结束。
  - 触发结构服务、日志与可视化：
    - 将轨迹与统计信息交给 `StructureService`；
    - 调用日志与可视化的回调。
  - 根据配置决定训练长度、评估频率等策略。
- **为什么重要**：
  - 统一的 Trainer 让架构更清晰，也使得替换 Agent / 环境时无需改动控制流程。

---

### 5. Episode（回合 / 试次）

- **含义**：
  - 从环境重置（起点状态）开始，到满足终止条件（如到达终点、步数上限）为止的一整条交互序列。
- **在本项目中的作用**：
  - Episode 是统计与分析的基本单元：
    - 每个 episode 产生一条路径（或失败记录）；
    - 可以计算回报、路径长度、是否成功等指标。
  - 结构服务通常以 episode 为粒度接收信息，以此构建更高层次结构。

---

### 6. 结构服务（Structure Service）

- **含义**：
  - 一个专门用于**接收和管理结构性信息**的服务模块，而不是具体算法或模型本身。
- **在本项目中的作用**：
  - 从 Trainer 接收轨迹（状态、动作、奖励等），并进行整理与归纳。
  - 构建和维护某种形式的“内部结构表示”，例如：
    - 常见路径集合；
    - 子目标或关键节点；
    - 宏动作或路径片段。
  - 提供接口用于：
    - 导出结构数据（供分析或可视化）；
    - （可选）向 Agent 提供结构性信息（例如推荐宏动作）。
- **边界**：
  - 不直接负责优化参数或控制训练；
  - 尽量与具体 Agent / 环境解耦，通过统一接口交互。

---

### 7. 路径记忆（Path Memory）

- **含义**：
  - 对智能体在环境中走过的路径（state / action 序列）的记录与存储。
- **与一般“经验缓冲”的区别**：
  - 在这里更强调**完整路径**或**片段**作为结构的候选单元，而不仅是单步转移。
- **在本项目中的作用**：
  - 作为 `StructureService` 的基础输入：
    - 分析哪些路径是成功的、哪些是常见的；
    - 比较不同 episode 中的路径相似性与差异。
  - 后续可能被用来：
    - 提取宏动作；
    - 构建子目标图或通路图。

---

### 8. 宏（Macro）

- **含义**：
  - 将一段连续的基本动作序列视为一个更高层次的“宏动作”（macro action / option）。
  - 例如：在栅格地图上，“向右走 3 步再向上走 2 步”可以被封装为一个宏。
- **在本项目中的作用（规划）**：
  - 作为“最小可复用结构”的候选形式之一：
    - 通过对成功路径的分析提取宏；
    - 在决策时允许 Agent 调用宏而不是只用一步步基本动作。
  - 有助于研究：
    - Agent 是否会偏好某些宏；
    - 宏的结构是否可解释（例如是否对应明显的走廊 / 通路）。

---

### 9. 发现（Discovery）

- **含义**：
  - 在训练或探索过程中自动识别新的模式、结构或宏，而非人工事先定义。
- **在本项目中的具体含义**：
  - 在 2D 最短路径任务中：
    - 从大量路径与失败案例中“发现”常见的子路径或关键节点；
    - 自动构建候选宏或子目标图；
    - 观察这些结构如何随训练进展而变化。
- **与“手工设计结构”的区别**：
  - 重点在于**涌现**与**自动发现**，而不是完全由研究者指定结构。

---

### 10. 日志 / 指标（Logging / Metrics）

- **含义**：
  - 记录训练 / 采样过程中产生的各种信息，用于调试、分析和复现。
- **常见内容**：
  - 标量指标：
    - 平均回报、成功率、平均路径长度；
    - episode 时长、重试次数等。
  - 结构性指标（规划）：
    - 宏数量、常见路径覆盖度；
    - 子目标节点数量等。
  - 调试信息：
    - 出错栅格地图；
    - 典型失败路径等。
- **在架构中的位置**：
  - 以回调 / 观察者的形式接入 `Trainer`；
  - 不参与决策，只消费数据。

---

### 小结

以上概念为本项目的“术语表”，用于帮助未来的自己或合作者快速建立共同语境。  
在后续迭代中，如果引入新的结构形式或实验设定，应在此文档中同步补充或调整对应定义。
