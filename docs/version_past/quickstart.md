## 快速上手（规划版）

本快速上手指南描述了**未来预期的使用方式**，帮助新用户 / 开发者理解如何运行第一个实验。  
目前大部分内容仍处于“规划”阶段，具体命令与模块名称以实际实现为准。

---

### 1. 前置条件

- **Python 环境**
  - 建议使用 Python 3.9+（具体版本以 `requirements.txt` 为准）。
- **操作系统**
  - 目标支持：Linux / macOS / Windows（当前开发环境以你本机为主）。

---

### 2. 克隆仓库与安装依赖（规划）

```bash
git clone <your-repo-url> lab_v2
cd lab_v2
```

建议使用虚拟环境（示例）：

```bash
python -m venv .venv

# Windows PowerShell
.venv\Scripts\Activate.ps1

# Linux / macOS
source .venv/bin/activate

pip install -r requirements.txt
```

> 说明：`requirements.txt` 的具体内容将在实现过程中补充，目前尚未固定。

---

### 3. 预期的目录与模块（只读理解）

在完成 v0 后，预期会存在以下关键目录（名称可能会有微调）：

- `envs/`：2D 栅格环境和地图生成器。
- `agents/`：不同 Agent 实现。
- `structure/`：结构信息记录模块和结构数据类型。
- `trainer/`：训练循环控制。
- `logging/`：日志与指标记录。
- `viz/`：可视化工具。
- `cli/`：命令行入口脚本。

目前如果这些目录还不存在，属于**尚未实现**，请以实际仓库结构为准。

### 4. 首个实验的预期工作流（规划）

以下是 v0 完成后预期的“第一个可运行实验”流程。  
在实现完成前，这些命令**尚不可用**，仅作为设计目标。

#### 4.1 运行最简单的栅格实验

计划提供一个命令行入口，例如：

```bash
python -m cli.run_simple_grid_experiment \
  --episodes 100 \
  --map-size 16 \
  --seed 42
```

预期行为（规划）：

- 创建一个 16x16 的栅格地图，包含起点、终点和若干障碍。
- 使用默认的基础 Agent（例如随机或简单启发式）在环境中运行 100 个 episode。
- 在终端输出：
  - 每 N 个 episode 的平均回报 / 成功率；
  - 平均路径长度等基础统计。

#### 4.2 查看结构相关输出

在“结构观察版本”（里程碑 3）之后，运行上述命令后，应能看到：

- 在某个输出目录（例如 `outputs/` 或 `runs/`，名称待定）中：
  - 保存每个 episode 的路径记录（例如 JSON / CSV / 简单文本）。
  - 保存若干结构摘要（例如最常见成功路径、成功率统计等）。
- 可选：在终端打印一两条代表性路径的栅格文本可视化。

示意（规划）：

```text
Episode 10: success=True, steps=23
###########
#S....#...#
#.###.#.#G#
#.....#...#
###########
Path: UURR... (略)
```

> 具体格式将在实现时确定，这里只是示例。

### 5. 开发者向：如何添加自己的实验（规划）

在基础框架稳定后，期望支持如下开发流程：

1. **定义或选择环境**（在 `envs/` 中）：
   - 使用已有的 2D 栅格环境；
   - 或复制一个基础环境模板并做小改动。
2. **选择或实现 Agent**（在 `agents/` 中）：
   - 实现符合统一接口的 Agent 类；
   - 在 CLI 中添加一个参数，用于选择不同 Agent。
3. **通过 Trainer 组合**：
   - 在 `trainer/` 中编写新的训练脚本（或在现有 Trainer 上增加配置）；
   - 挂载结构记录、日志和可视化回调。
4. **添加 CLI 入口**：
   - 在 `cli/` 中添加一个新的命令或子命令，指向某个 Trainer 配置。

> 上述步骤目前都是规划，具体 API 以代码实现为准。
