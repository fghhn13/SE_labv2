### 代码与架构进展

- **核心训练闭环**
  - 实现并连通：

    - `RandomAgent` / `GreedyAgent`（Agent）

- **Agent 体系**
  - `RandomAgent`：作为最简单 baseline。
  - `GreedyAgent`：
    - 基于曼哈顿距离的贪心策略；
    - 支持 `epsilon` 探索；
    - 通过注册表统一由配置驱动。

- **环境与地图**
  - 引入 `GridMap` + `GridWorldEnvironment` 的清晰分工。
  - 新增 `lab.envs.grid.maps` 作为**统一地图注册与生成入口**：
    - 内置地图：`open_5x5`、`random_blocks_10x10`。
  - 实验配置只写 `env.map_name`，不再内嵌具体宽高 / 起终点 / 墙的细节。

### 日志与可视化

- **实验结果组织**
  - 统一使用：
    - `lab_v2_results/<agent_name>/<map_name>/...`
  - 避免不同模型 / 地图的结果互相覆盖，方便后续对比。

- **每次训练结束自动产出**
  - `*.csv`：每 episode 的 `success` / `cost` / `steps`。
  - `maps/*.png`：
    - 代表性路径图（最频繁 / 最好 / 最坏）。
    - 底图编码：空白 / 墙 / 起点 / 终点，用不同颜色表示。
  - `plots/*.png`：
    - 成功率（滑动平均）曲线；
    - 步数 / cost 曲线（双子图）。
  - 所有图表均可选依赖 `.[viz]`（matplotlib + pandas），未安装时训练不受影响。

### 实验配置与脚本

- **experiments 目录重构**
  - 按 Agent 名称拆分：
    - `experiments/random/...`
    - `experiments/greedy/...`
  - 示例：
    - `basic_grid_open_5x5.json`
    - `basic_grid_10x10_random_blocks.json`

- **一键脚本**
  - `lab.cli.run_experiment`：
    - 支持断点续训；

  - `lab.cli.run_and_plot`：
    - 在 `run_experiment` 基础上，单独支持“先训练再画图”的批处理场景。
